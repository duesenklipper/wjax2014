Was ist Wicket?

  • Java-Web-Framework
  • Reines HTML-Template
  • Alle Logik in Java (oder Scala, ...)
evtl kurze demo

Warum heute noch Wicket?

Wicket ist 10 Jahre alt Ist das überhaupt noch modern?

Warum überhaupt noch server-side?

  • Weniger Last auf dem Client
  • Validierung und Security
  • Kompilierte Sprachen
  • Bessere Trennung von Code und Markup
clientseitig: interaktiver, schicker
kompilierte sprachen -> nicht unbedingt ein problem, aber geschmackssache
html + js ähnelt JSP-tag-und-code-suppe

Warum dann Wicket?

  • Bewährtes Komponentenmodell
  • Sauberes HTML-Templating
  • Direkte Kontrolle über Output
  • Sauberes State Management
  • Kleine, klare Interfaces
  • Gut automatisiert testbar
es bietet sich an wicket für die grundlagen zu nehmen, und mit client-framework zu "veredeln"

Wicket ist HTML-Templating und State Management

Was ist neu in Wicket 7?

...viele Kleinigkeiten.

  • Java 7, Servlet 3
  • API aufgeräumt
    • Viel deprecated Code entfernt
    • Unglückliche Methodensignaturen verbessert
  • CSS durch Properties konfigurierbar
  • Und noch viel mehr: https://cwiki.apache.org/confluence/display/WICKET/Migration+to+Wicket+7.0
